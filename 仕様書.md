# 見積書自動生成システム 実装仕様書

## 1. 背景・課題

- 別システムから抽出した実績データ(出勤表など)をスプレッドシートにまとめ、目視で必要項目を拾って手作業で見積書に転記している
- 部署・営業マンごとにExcelテンプレートが異なり、新人が迷う
- 目視転記による作業時間の長さ・入力ミスのリスクが課題

## 2. ゴール

出勤実績・顧客情報から、ボタン一つで統一フォーマットの見積書が自動生成される状態にする。
人間は「内容を確認するだけ」になり、作業時間を数時間→数分に短縮する。

---

## 3. データ構成

### 3.1 入力データ(ダミーデータとしてスプレッドシートに用意)

**Customers シート**(顧客マスタ、1社1行)

| カラム名 | 内容 | 例 |
|---|---|---|
| customer_code | 顧客コード(キー) | C001 |
| customer_name | 顧客名 | Alpha Dental Clinic |
| contact_person | 担当者名 | Sato |
| unit_price | 単価(固定月額) | 120000 |
| tax_rate | 消費税率(%) | 10 |
| quote_template | テンプレート種別 | standard / executive |

**Attendance シート**(出勤実績、1社につき複数行あり得る)

| カラム名 | 内容 | 例 |
|---|---|---|
| customer_code | 顧客コード(キー) | C001 |
| work_month | 対象月 | 2026-04 |
| staff_name | スタッフ名 | Yamada |
| attendance_days | 出勤日数 | 18 |
| work_hours | 勤務時間 | 144 |

### 3.2 データの紐付け

`customer_code` をキーに、Customers(1件)に対してAttendanceが複数件ヒットする1対多の関係。
Attendanceは「当該月に実績があるかどうか」のフィルタ条件として使用し、**金額計算そのものには時間数を使わない**(unit_priceは固定月額のため)。

---

## 4. 金額計算ロジック

```
金額         = unit_price (固定月額、複数スタッフがいても合算しない)
消費税額     = 金額 × tax_rate / 100
合計金額     = 金額 + 消費税額
```

- 1顧客に複数スタッフがいる場合も、見積書には**合計のみ表示**(スタッフ名は出さない)
- Attendanceに対象月のデータが1件もない顧客は、見積書作成の対象外とする

---

## 5. シート構成

```
Customers シート     … 顧客マスタ(ダミーデータ)
Attendance シート    … 出勤実績(ダミーデータ)
QuoteFormat シート   … 見積書レイアウト(1枚を使い回す)
```

### 5.1 QuoteFormat シートのレイアウト案

| セル | 内容 |
|---|---|
| B3 | 顧客名 |
| B4 | ご担当者名 |
| B5 | 対象月 |
| B9 | 金額(税抜) |
| B10 | 消費税額 |
| B11 | 合計金額 |

`quote_template` が `standard` / `executive` の2種類あるため、将来的にレイアウトを出し分ける拡張性を考慮した設計とする(初期実装では1種類のレイアウトで開始し、後で分岐を追加してもよい)。

---

## 6. 機能要件

1. カスタムメニュー「見積書作成」から実行できること
2. ボタン一つで**全顧客分を一括生成**すること(顧客選択ダイアログは設けない)
3. QuoteFormatシートに、顧客ごとのデータを書き込んで使い回すこと(顧客ごとにシートを複製しない)
4. 可能であれば、QuoteFormatシートの内容をPDF化してGoogle Driveに保存すること

---

## 7. エラーハンドリング要件(最低限)

- 必須項目(顧客名・金額など)が空欄の場合、アラートを表示して処理を中断する
- アラート文言:「必須項目が入力されていません」
- 対象:`customer_name`, `unit_price` など、見積書に必須の項目

## 9. 未確定・実装時に決める項目

- 対象月(`work_month`)の指定方法:ダイアログ入力 or Attendance内の最新月を自動検出
- PDFの保存先フォルダ(Drive内の特定フォルダ/スプレッドシートと同じフォルダ)
- 見積書の発行者(自社)情報・見積番号の要否
- エラー時、1件スキップして次の顧客へ進むか、全体を中断するか
