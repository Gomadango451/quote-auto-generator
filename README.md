# スプレッドシートのデータを利用した「見積書自動作成」

## 1. 要件定義
- 解決したい業務課題: 別システムから抽出した出勤実績を毎回目視で確認し、手作業で見積書に転記している。部署・担当者ごとにテンプレートが異なり、新人が迷う状況も発生していた
- 想定利用者: 見積書の作成・送付を担当する営業・管理部門のスタッフ
- 期待する業務効果: 全顧客分の見積書作成にかかる時間を数時間から数分に短縮し、手入力による転記ミスをゼロにする

## 2. 実装概要
- 課題の目的: 出勤実績と顧客マスタから、ボタン一つで統一フォーマットの見積書PDFを自動生成する
- 実装したもの: Google Apps Script による見積書自動生成システム。カスタムメニューから実行し、全顧客分の見積書をPDF化してGoogle Driveに一括保存する
- 利用した技術: Google Apps Script / SpreadsheetApp API / DriveApp API / UrlFetchApp（Sheets Export API）

## 3. 機能要件への対応
- 顧客情報・金額データから別シートまたは PDF へ見積書フォーマットで転記されること: Customers シートの顧客情報（顧客名・担当者・単価・税率）と Attendance シートの出勤実績を結合し、QuoteFormat シートに転記したうえでPDF化してDriveに保存する
- GAS のカスタムメニューから実行できること: `onOpen()` によりスプレッドシートを開くと「見積書作成」メニューが追加され、「全顧客分を生成」から実行できる

## 4. 設計
- データの流れ: Customers シート（顧客マスタ）と Attendance シート（出勤実績）を `customer_code` で紐付け → QuoteFormat シートにレイアウトを構築 → Sheets Export API でPDF変換 → Google Driveの「見積書」フォルダに保存
- 画面または処理の構成:
  - `onOpen()` … カスタムメニューを追加
  - `generateAllQuotes()` … メイン処理。対象月をダイアログで取得し全顧客をループ
  - `getCustomers()` / `getAllAttendance()` … 各シートを1回だけ読み込んでオブジェクト配列に変換
  - `filterAttendance()` … 読み込み済みデータをメモリ上でフィルタ（API呼び出し削減）
  - `validateRequiredFields()` … 必須項目の空欄チェック
  - `fillQuoteFormat()` … シートをクリアしてレイアウト・スタイルを毎回構築
  - `exportSheetAsPdf()` … PDF化とDrive保存。429エラー時は最大3回リトライ
  - `getOrCreatePdfFolder()` … 保存先フォルダの取得または自動作成
- 工夫した設計判断:
  - Attendance の読み込みをループ外で1回だけ行い、ループ内はメモリ上でフィルタすることでAPI呼び出し回数を削減した
  - 自社情報・シート名・行番号をファイル上部に定数として集約し、変更箇所を1か所に限定した
  - 列番号はヘッダー名で動的に取得（`headers.indexOf()`）し、列の追加・並び替えに対応できる設計にした
  - 出勤スタッフ数が顧客ごとに異なるため、固定テンプレートではなくコードでレイアウトを毎回構築する方式を採用した
  - PDF出力前に `SpreadsheetApp.flush()` を呼び、シートへの書き込みが確定してからエクスポートされるようにした

## 5. エラーハンドリング
- 顧客名や金額などの必須項目が空欄の場合にアラートを表示すること: `validateRequiredFields()` で `customer_name` と `unit_price` の空欄・null を確認し、いずれかが欠けている場合はアラートを表示して処理を停止する
- エラー時は「必須項目が入力されていません」と表示して処理を中断すること: `ui.alert('必須項目が入力されていません')` を表示後、`return` で `generateAllQuotes()` を即時終了し、以降の顧客処理を行わない

## 8. 実行手順

### 事前準備
- Google アカウントが必要です
- Google Drive 上でスプレッドシートを作成してください（ローカルの Excel では動作しません）

### CSVのデータ形式

**Customers シート**（1行目はヘッダー行）

| カラム名 | 型 | 説明 | 例 |
|---|---|---|---|
| customer_code | 文字列 | 顧客コード（一意のキー） | C001 |
| customer_name | 文字列 | 顧客名（必須） | Alpha Dental Clinic |
| contact_person | 文字列 | 担当者名 | Sato |
| unit_price | 数値 | 月額固定単価（必須） | 120000 |
| tax_rate | 数値 | 消費税率（%） | 10 |
| quote_template | 文字列 | テンプレート種別 | standard |

**Attendance シート**（1行目はヘッダー行）

| カラム名 | 型 | 説明 | 例 |
|---|---|---|---|
| customer_code | 文字列 | 顧客コード（Customers と紐付け） | C001 |
| work_month | 文字列 | 対象月（YYYY-MM形式） | 2026-04 |
| staff_name | 文字列 | スタッフ名 | Yamada |
| attendance_days | 数値 | 出勤日数 | 18 |
| work_hours | 数値 | 勤務時間 | 144 |

> **注意**: カラム名は上記と完全に一致させてください。順番は問いません。

### セットアップ手順
1. Google スプレッドシートを新規作成し、以下の3シートを用意する
   - `Customers` … 上記形式のデータを用意してインポート（ファイル → インポート → アップロード → 現在のシートに置き換える）
   - `Attendance` … 上記形式のデータを用意してインポート（同上）
   - `QuoteFormat` … 空のシートを作成（GASが自動で構築するため、内容は不要）
2. 拡張機能 → Apps Script を開き、`Code.gs` の内容を貼り付けて保存する（Ctrl+S）
3. スプレッドシートをリロードすると「見積書作成」メニューが追加される
4. 見積書作成 → 全顧客分を生成 をクリックし、対象月（例: `2026-04`）を入力して実行する
5. 初回実行時は Google から権限の確認ダイアログが表示される。「詳細」→「安全でないページに移動」→「許可」の順にクリックする
6. スプレッドシートと同じフォルダ内に「見積書」フォルダが自動作成され、PDFが保存される

### カスタマイズ方法
`Code.gs` の冒頭にある定数を変更するだけで自社情報や設定を変更できます。

```javascript
// 自社情報（見積書の右上に表示される）
const COMPANY_NAME     = '株式会社〇〇〇〇';   // ← 自社名に変更
const COMPANY_ZIP      = '〒000-0000';          // ← 郵便番号に変更
const COMPANY_ADDRESS  = '都道府県市区町村番地'; // ← 住所に変更
const COMPANY_BUILDING = 'ビル名 階数';          // ← ビル名に変更（不要なら空文字 '' にする）

// シート名（スプレッドシートのシート名と一致させること）
const SHEET_CUSTOMERS  = 'Customers';
const SHEET_ATTENDANCE = 'Attendance';
const SHEET_QUOTE      = 'QuoteFormat';

// PDF保存フォルダ名
const PDF_FOLDER_NAME  = '見積書';
```

## 9. ビジネス価値
- 転記ミスの排除: 顧客マスタと出勤実績から自動で値を取得・計算するため、手入力による金額ミスや顧客名の誤りが発生しない
- 作業時間を数時間 → 数分に短縮: 対象月を入力するだけで全顧客分のPDFが一括生成され、個別の手作業が不要になる
- フォーマットの統一: QuoteFormat シートを単一のマスターとして使い回すため、担当者や部署が変わっても同一フォーマットの見積書が出力される
- 確認業務への集中: 人間の作業を「内容の最終確認」のみに絞ることができ、より付加価値の高い業務に時間を使えるようになる

## 10. 今後の改善点
- `quote_template`（standard / executive）に応じたレイアウトの出し分けを実装する
- 対象月をダイアログ入力ではなく Attendance 内の最新月から自動検出する
- 見積番号の採番ルールを管理シートで一元管理し、重複を防ぐ
- エラー発生時に全体中断ではなく該当顧客をスキップして続行するモードを追加する
